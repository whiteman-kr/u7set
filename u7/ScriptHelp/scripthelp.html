General Information

Using scripts, project designer is able to customize displayed schemas. Scripts allow to change colors, text messages and other appearance properties of schema items. Also scripts can read signal values, set tuning signal values and perform some additional actions.

Scripts are written in JavaScript language.

Every schema item is an object that has a set of properties. These properties can be changed manually in Schema Editor or by script at the run time. For example, properties can change text, foreground and background color. Different items have special properties.

Schema items have scripts that executed at certain events: before drawing, at mouse click, etc. For example, all schema items have PreDrawScript, ClickScript events. Different items have special events.

When script is called, some parameters are passed to it. For example, script function always has a handle to processed schema item.

To access random schema items, variables and to switch between schemas, global SchemaView object is provided.

To access signal values that are received from Application Data Service, ScriptAppSignalController object is provided.

To access and set tunable signal values using Tuning Service, TuninglController object is provided.

Global variables

SchemaView

This object is accessed by view object name and is used for switching schemas, searching items and widgets and displaying message boxes.

 

Functions

 

setSchema(schemaId : string) : void;

Sets the active schema specified in schemaId parameter.

 

findSchemaItem(objectName : string) : object;

Finds a schema item by its name (ObjectName property). Return value has SchemaItem type or undefined if item is not found.

 

findWidget(objectName : string) : object;

Finds a schema control widget (edit control, button, etc...) by its name (ObjectName property). Return value has QWidget type or undefined if item is not found.

 

warningMessageBox(text : string) : void;

Displays a warning message box with specified text.

 

errorMessageBox(text : string) : void;

Displays an error message box with specified text.

 

infoMessageBox(text : string) : void;

Displays an information message box with specified text.

 

questionMessageBox(text : string) : boolean;

Displays a question message box with specified text. If user clicked "Yes", returns true, otherwise returns false.

 

variableExists(QString name) : boolean;

Returns true if variable specified by name exists, otherwise returns false.

 

variable(QString name) : string;

Retrieves a value of the variable specified by name.

 

setVariable(name : string, value : string) : void;

Sets the value of the variable specified by name.

 

Example

view.setSchema("MYSCHEMA");

view.infoMessageBox(“Hello world!”);

if (view.questionMessageBox(“Are you sure?”) == true)

{

    ....

}

 

ScriptAppSignalController

 

This object is accessed by signals object name and is used to get signals parameters and values in Monitor application.

 

 Functions

 

signalParam(signalId : string) : object;

Returns AppSignalParam structure of signal specified by signalId. If error occurs, the return value is undefined.

 

signalState(signalId : string) : object;

Returns AppSignalState structure of signal specified by signalId. If error occurs, the return value is undefined.

 

Example

var sp = signals.signalParam("#SIGNALID_001");

var st = signals.signalState("SIGNALID_001");

if  (sp == undefined || st == undefined)

{

    return;

}

 

TuningController

 

This object is accessed by tuning object name and is used to get tuning signals parameters and states, write tuning values and in Monitor and TuningClient applications.

 

Functions

 

signalParam(appSignalID : string) : object;

Returns AppSignalParam structure or undefined if signal does not exist.

 

signalState(appSignalID : string) : object;

Returns TuningSignalState structure or undefined if signal does not exist.

 

writeValue(appSignalID : string, value : number) : boolean;

Writes value of tuning signal. If successful, returns true, otherwise returns false.

 

Example

tuning.writeValue("SIGNALID", v);

 

Schema Items

 

Setting properties of a schema item

 

To set a property value of a schema item, use the following code:

 

schemaItem.setPropertyValue("PropertyName", PropertyValue);

 or

schemaItem.PropertyName = PropertyValue;

 

To view full set of properties of a schema item, open the Proprties window in Schema Editor in RPCT.

 

SchemaItem

 

SchemaItem is a base class for all items that are displayed on schemas.

 

Essential Properties

 

ObjectName (string)

A name of an item. Must be unique on the schema.

 

AcceptClick (bool)

When set to true, mouse cursor is displayed as a "hand" above an item and ClickScript is executed on the left mouse button click.

 

Scripts

 

ClickScript

A script that is executed when user clicks the left mouse button on the item.

Parameters:

schemaItem - the processed schema item. Object type: SchemaItem

 

Example

(function(schemaItem)

{

    view.setSchema("SCHEMA002");

})

 

PreDrawScript

A script that is executed before item is drawn.

Parameters:

SchemaItemValue

 

This item is used to display signal values. To customize its behaivour, PreDrawScript and ClickScript are used.

 

Accessing signals identifiers

To access signal identifiers of this item, use the following code:

 

var appSignalId = schemaItemValue.SignalIDs[0];

 

Essential Properties

 

DrawRect (boolean)

Draw the border

 

FillColor (string)

Rectangle fill color name

 

LineColor (string)

Border line color name

 

LineWeight (int)

Border line weight, in pixels

 

TextColor (string)

Text color name

 

AlignHorz (enum)

Horizontal text alignment

 

AlignVert (enum)

Vertical text alignment

 

FontBold (boolean)

Bold font

 

FontItalic (boolean)

Italic font

 

FontName (string)

Font name

 

FontSize (int)

Font size

 

Text (string)

Rectangle text, can contain macros:

 

$(value) - signal value

$(caption) - signal caption

$(signalid) - custom signal identifier

$(appsignalid) - signal identifier

$(equipmentid) - signal equipment id

 

AppSignalIDs (string array)

Application identifiers for an item

 

Precision (int)

Numbers precision

 

Signal Source (enum)

Determies the default source of signal values - ApplicationDataService or TuningService

 

Example of PreDrawScript for SchemaItemValue

(function(schemaItemValue)

{

    var appSignalId = schemaItemValue.SignalIDs[0];

 

    // Get data from TuningService

    var signalParam = tuning.signalParam(appSignalId);

    var signalState = tuning.signalState(appSignalId);

 

    // Get signal state

    if (signalState.Valid == true)

    {

        schemaItemValue.Text = signalState.Value;

        schemaItemValue.TextColor = "white";

        schemaItemValue.FillColor = schemaItemValue.BlinkPhase ? "black" : "#A00000";

        schemaItemValue.LineColor = "#000000";

    }

})

SchemaItemLineEdit

 

This control is used to create user input fields.

 

Essential Properties:

 

AlignHorz (enum)

Horizontal text alignment

 

AlignVert (enum)

Vertical text alignment

 

MaxLength (int)

Maximum text length

 

PlaceholderText (string)

Property holds the line edit's placeholder text. Setting this property makes the line edit display a grayed-out placeholder text as long as the line edit is empty. Normally, an empty line edit shows the placeholder text even when it has focus. However, if the content is horizontally centered, the placeholder text is not displayed under the cursor when the line edit has focus. By default, this property contains an empty string.

 

StyleSheet (string)

A stylesheet of a widget. Detailed description of this field can be found in Qt documentation.

 

Text (string)

Property holds the line edit's text.

 

ReadOnly (boolean)

This property holds whether the line edit is read only. In read-only mode, the user can still copy the text to the clipboard, or drag and drop the text, but cannot edit it. LineEdit does not show a cursor in read-only mode.

 

Tooltip (string)

Gets or sets the tooltip text in the control.

Describes signals static parameters.

AppSignalID: string;

CustomSignalID: string;

Caption: string;

EquipmentID: string;

Unit: string;

Precision: number;

Channel: number;

EnableTuning: boolean;

TuningDefaultValue: boolean;

TuningLowBound: number;

TuningHighBound: number;

IsInput: boolean;

IsOutput: boolean;

IsInternal: boolean;

IsAnalog: boolean;

AppSignalState

 

Describes signal state in the Monitor application, is received from Application Data Service.

 

Properties

Hash: 64-bit number;

Value: number;

Valid: boolean;

StateAvailable: boolean;

Simulated: boolean;

Blocked: boolean;

Unbalanced: boolean;

AboveHighLimit: boolean;

BelowLowLimit: boolean;

OutOfLimits: boolean;

 

TuningSignalState

 

Describes tuning signal state in the Monitor and TuningClient applications, is received from TuningService. Can be used in Monitor only if Tuning feature is turned on.

 

Properties

Value: number;

HighBound: number;

Valid: boolean;

OutOfRange: boolean;

WriteInProgress: boolean;

WritingIsEnabled: boolean;
